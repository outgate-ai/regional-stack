DETECTION CRITERIA: Authentication Credentials and Secrets

Detect the following types of credentials:
- API keys (sk-, pk-, api_, Bearer tokens)
- Passwords and passphrases
- Access tokens and refresh tokens
- Private keys and certificates
- Database connection strings
- OAuth tokens and secrets

EXTRACTION RULES:
- For KEY=VALUE patterns, extract ONLY the value (after the = sign)
- For connection strings (e.g., postgresql://user:pass@host), extract BOTH the full string AND the password separately
- For Bearer tokens, extract only the token value (not the "Bearer" prefix)
- Never include the key name or assignment operator in the extracted text

COMPONENT EXTRACTION:
When a credential contains embedded sub-credentials, extract both the whole and the sensitive parts:
- "postgresql://admin:mySecret@host/db" → extract "postgresql://admin:mySecret@host/db" AND "mySecret"
- "mongodb+srv://root:p4ssw0rd@cluster.mongodb.net" → extract the full URL AND "p4ssw0rd"
- "redis://:tokenXyz@redis.host:6379" → extract the full URL AND "tokenXyz"
This ensures individual secrets are detected even when they appear outside the original connection string.

EXAMPLES:
- Input:
  <content_to_analyze>
  [user] Use API key sk-1234567890abcdef to authenticate
  </content_to_analyze>
  Output: [{"text": "sk-1234567890abcdef", "category": "credentials"}]

- Input:
  <content_to_analyze>
  [user] POSTGRES_PASSWORD=adjkfh98hf
  DATABASE_URL=postgresql://dbuser:adjkfh98hf@postgres:5432/mydb
  INTERNAL_API_KEY=internal_key_replace_me_in_production
  VITE_GOOGLE_CLIENT_ID=38726138r7ry182476r876-cbkjhdbckushdjbvcakuycbeurycbajkshdbc.apps.googleusercontent.com
  </content_to_analyze>
  Output: [{"text": "adjkfh98hf", "category": "credentials"}, {"text": "postgresql://dbuser:adjkfh98hf@postgres:5432/mydb", "category": "credentials"}, {"text": "internal_key_replace_me_in_production", "category": "credentials"}, {"text": "38726138r7ry182476r876-cbkjhdbckushdjbvcakuycbeurycbajkshdbc.apps.googleusercontent.com", "category": "credentials"}]

- Input:
  <content_to_analyze>
  [user] My password is Secret123! for login
  </content_to_analyze>
  Output: [{"text": "Secret123!", "category": "credentials"}]
