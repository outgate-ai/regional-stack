DETECTION CRITERIA: Authentication Credentials and Secrets

Detect the following types of credentials:
- API keys (sk-, pk-, api_, Bearer tokens)
- Passwords and passphrases
- Access tokens and refresh tokens
- Private keys and certificates
- OAuth tokens and secrets
- Secret values in KEY=VALUE environment variables

EXTRACTION RULES — extract ONLY the secret value itself:
- For KEY=VALUE patterns, extract ONLY the value (after the = sign)
- For Bearer tokens, extract only the token value (not the "Bearer" prefix)
- Never include the key name, assignment operator, or structural syntax

CONNECTION STRINGS — extract ONLY the embedded secrets, NOT the full URL:
- "postgresql://admin:mySecret@host/db" → extract "mySecret" (the password only)
- "mongodb+srv://root:p4ssw0rd@cluster.mongodb.net" → extract "p4ssw0rd"
- "redis://:tokenXyz@redis.host:6379" → extract "tokenXyz"
- Do NOT extract the host, port, database name, or protocol — these are infrastructure metadata, not credentials
- Do NOT extract generic usernames like admin, root, postgres, sa — only extract passwords/tokens

This ensures the actual secret is detected independently wherever it appears (in env vars, config files, code, or URLs).

  </content_to_analyze>
  Output: [{"text": "sk-1234567890abcdef", "category": "credentials"}]
